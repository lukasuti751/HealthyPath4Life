About HealthyPath4Life

HealthyPath4Life is the web interface for the Veg3n on-chain companion. Veg3n lets you log meals (as hashes), set nutrition goals, join time-bound paths and earn points. The path coordinator creates paths; the reward keeper awards points. All data lives on the EVM chain.

The companion is designed for an AI healthy eating lifestyle: you record what you eat (as a hash for privacy or consistency), tag meals with path tags (e.g. balanced, plant-based), and optionally join community paths. Points and daily streaks encourage consistency. Goals let you set a target (stored as a hash and value) for your nutrition journey.

This site does not store your meal data on a server; hashing is done in your wallet or in the v3gn CLI. The contract stores only hashes and metadata on-chain.

Meal logging

Log a meal with a meal hash, path tag and type (breakfast, lunch, dinner, snack). Optionally batch log.

Paths

Paths are time-bound (start/end block). Join a path to track your journey with others.

Points & goals

Earn points when the reward keeper awards them. Set a goal (hash + target value) for your nutrition target.

Paths & tips

Paths are created by the path coordinator. You can join or leave while a path is active. Tips (AI healthy eating tips) are recorded as hashes by the coordinator.

Meal types: 1 = Breakfast, 2 = Lunch, 3 = Dinner, 4 = Snack. Hash your meal data off-chain (e.g. keccak256) before logging.

Meal types reference

  • 1 — Breakfast
  • 2 — Lunch
  • 3 — Dinner
  • 4 — Snack

Breakfast (1)

Log morning meals: oatmeal, eggs, fruit, smoothies, etc. Use mealHash = keccak256 of your meal description.

Lunch (2)

Midday meals. Combine with a path tag (e.g. "balanced", "plant-based") for tracking.

Dinner (3)

Evening meals. Same hashing and path tag logic as other types.

Snack (4)

Snacks between meals. Helps build a full picture of your eating pattern.

Path lifecycle

The path coordinator creates a path with createPath(pathTag, startBlock, endBlock). Paths are identified by a sequential pathId. While the current block is between startBlock and endBlock, the path is active. Users call joinPath(pathId) to participate and leavePath(pathId) to leave. Path participant count is updated automatically. Use getActivePathIds() to list currently active paths.

FAQ

        

          
What is Veg3n?

          
Veg3n is an on-chain AI healthy eating companion. You log meals (as hashes), set goals, join paths and earn points. Path coordinator and reward keeper are set at deployment.

          
How do I log a meal?

          
Call logMeal(mealHash, pathTag, mealType). Hash your meal description off-chain (e.g. keccak256). Meal type: 1=breakfast, 2=lunch, 3=dinner, 4=snack.

          
What are paths?

          
Paths are time-bound (start block, end block). The path coordinator creates them. You join with joinPath(pathId) while the path is active.

          
How do I get points?

          
The reward keeper awards points via awardPoints or awardPointsForMeal. You can redeem points with redeemPoints(amount, reasonHash).
